Notify Admin

When enough training pairs have been collected, click the button below to notify the admin that a fine-tuning run can be started. The notification is recorded and, if email is configured on the server, sent automatically.

This guide targets the current server setup: Qwen2.5-VL 7B, NVIDIA V100 (16 GB), Ubuntu, uv-managed Python environment.

Step 1 — Export the training dataset

Run the export script from the project directory. It reads the archive and produces a JSONL file of usable (image, text) pairs, skipping jobs with no feedback or only weak signal.

cd /home/hermes/arabic-ocr
.venv/bin/python export_dataset.py --out dataset/arabic_ocr.jsonl

Verify the output before proceeding — open a few entries and confirm the image paths and corrected text look right.

Step 2 — Install LLaMA-Factory

cd ~
git clone https://github.com/hiyouga/LLaMA-Factory.git
cd LLaMA-Factory
pip install -e ".[torch,metrics]"

Step 3 — Register the dataset

Add an entry to LLaMA-Factory/data/dataset_info.json:

{
  "arabic_ocr_corrections": {
    "file_name": "/home/hermes/arabic-ocr/dataset/arabic_ocr.jsonl",
    "formatting": "sharegpt",
    "columns": { "messages": "conversations", "images": "images" }
  }
}

Step 4 — Run QLoRA fine-tuning

The V100 does not support bfloat16 — always use --fp16.
cd ~/LLaMA-Factory
llamafactory-cli train \
  --model_name_or_path sherif1313/Arabic-English-handwritten-OCR-v3 \
  --dataset arabic_ocr_corrections \
  --template qwen2_vl \
  --finetuning_type lora \
  --quantization_bit 4 \
  --output_dir /home/hermes/arabic-ocr/lora_output \
  --num_train_epochs 3 \
  --per_device_train_batch_size 1 \
  --gradient_accumulation_steps 4 \
  --learning_rate 1e-4 \
  --lora_rank 16 \
  --lora_target all \
  --save_steps 100 \
  --fp16

Step 5 — Evaluate before deploying

Hold out ~10 % of pairs before Step 1 and run inference on them with the new adapter to confirm CER (character error rate) improved. Do not deploy a model that scores worse on the held-out set.

Step 6 — Deploy the new model

Update MODEL_NAME in app.py to point to the local adapter output directory, then restart the service:

sudo systemctl restart arabic-ocr
journalctl -u arabic-ocr -f   # watch for load errors

The service will load the fine-tuned weights at startup. Monitor the first few jobs via the archive to confirm results improved.
